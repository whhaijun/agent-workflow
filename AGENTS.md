# AGENTS.md — 工作规范

## 启动流程

1. 读取 `IDENTITY.md`
2. 读取 `memory/hot.md`（HOT 层，≤100行）
3. 如果项目名匹配，读取 `memory/projects/[项目名].md`
4. 应用所有规则

## 任务执行流程

### WBS 评估（满足 2 条 → 必须拆分）
- 涉及文件 ≥ 3 个
- 步骤 ≥ 5 步
- 预计耗时 > 10 分钟

### 拆分原则
- 无依赖 → 并行（每批 ≤ 5 个）
- 有依赖 → 串行
- 每个子任务目标 ≤ 2 分钟

### Task Brief 格式
```
## 任务 ID: TB-YYYY-MM-DD-XXX
## 目标: [一句话]
## 标准: [可量化的完成条件]
## 交付物: [具体文件/结果]
```

## 学习流程

当用户说以下内容时，立即记录：
- "不对" / "错了" / "应该是" → 记录到 logs/YYYY/MM/DD.md
- "记得" / "总是" / "永远" → 记录到 memory/hot.md
- "我喜欢" / "我不喜欢" → 记录到 memory/hot.md

## 安全红线（绝对不能做）

1. 删除文件前必须二次确认
2. 不记录密码、API key、token
3. 不执行未验证的代码
4. 生产环境操作必须明确授权

## 错误处理

1. 工具失败 → 重试 3 次
2. 仍失败 → 尝试替代方案
3. 无替代 → 标注"⚠️ 跳过"继续
4. 最后汇总所有问题

## 质量自检（每次完成后）

- [ ] 是否达到 Task Brief 的完成标准？
- [ ] 是否有遗漏的边界/异常情况？
- [ ] 是否符合规范？
- [ ] 是否需要记录教训到 memory/hot.md？
